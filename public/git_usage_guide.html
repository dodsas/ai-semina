Git 사용 방법

실습에서 자주 쓰는 Git 주요 명령어를 업로드와 다운로드 흐름으로 정리한 자료입니다.

1

Git 주요 명령어

Git은 크게 업로드와 다운로드 흐름으로 이해하면 됩니다.

업로드

작성한 코드를 Git 저장소에 저장하고 서버에 업로드할 때 사용합니다.

파일 추가
git add .

현재 폴더의 변경된 파일을 Git에 추가합니다.

저장
git commit -m "작업 내용 작성"

추가된 파일을 하나의 작업 단위로 저장합니다.

git commit -m "회원가입 기능 추가"
서버에 업로드
git push

내 PC에 저장한 Git 변경사항을 원격 저장소 서버에 업로드합니다.

다운로드

서버에 있는 프로젝트를 내 PC로 다운로드하거나 최신 상태로 갱신할 때 사용합니다.

프로젝트 다운로드
git clone 저장소주소

원격 저장소에 있는 프로젝트를 내 PC로 복사합니다.

git clone https://github.com/example/project.git

여기서 “저장소주소”로 무엇을 넣어야 하는지는 아래 2번 — clone 주소 복사하기에서 자세히 설명합니다.

최신 내용 가져오기
git pull

이미 다운로드한 프로젝트에서 서버의 최신 변경사항을 다시 가져옵니다.

2

clone 주소 복사하기 (GitHub)

git clone 뒤에 넣는 주소는 웹 브라우저 주소창의 URL이 아니라, GitHub 저장소 페이지의 [Code] 버튼 안에 있는 저장소 주소(.git URL) 입니다. 아래 순서로 복사하세요.

① GitHub 저장소 페이지 열기

받아올 프로젝트의 GitHub 페이지로 이동합니다. 주소는 보통 https://github.com/사용자명/프로젝트명 형태입니다.

② 초록색 [Code] 버튼 클릭

파일 목록 위 오른쪽에 있는 초록색 <> Code 버튼을 누릅니다. 작은 창이 열리면 HTTPS 탭이 선택되어 있는지 확인합니다. (SSH / GitHub CLI 가 아니라 HTTPS 를 사용하세요.)

③ 주소 복사

칸에 표시된 주소 오른쪽의 복사 아이콘(📋) 을 눌러 주소를 복사합니다. 주소는 아래처럼 .git 으로 끝나는 형태입니다. 

https://github.com/사용자명/프로젝트명.git

④ 터미널에서 clone

VS Code 터미널에서 git clone 뒤에 복사한 주소를 붙여넣고 실행합니다. (붙여넣기: Ctrl + V) 

git clone https://github.com/사용자명/프로젝트명.git

실행하면 현재 터미널이 위치한 폴더 안에 프로젝트명과 같은 이름의 폴더가 새로 생기고, 그 안에 프로젝트 파일이 모두 다운로드됩니다.

다운로드 위치를 정하고 싶다면

clone 은 지금 터미널이 있는 폴더에 받습니다. 원하는 폴더로 먼저 이동한 뒤 clone 하세요. (cd = 폴더 이동) 

cd Desktop
git clone https://github.com/사용자명/프로젝트명.git
받은 프로젝트 폴더로 들어가기

clone 후에는 만들어진 폴더 안으로 이동해야 작업할 수 있습니다.

cd 프로젝트명

VS Code 라면 File > Open Folder 로 그 폴더를 열어도 됩니다.

주소를 못 찾겠다면 (가장 간단한 방법)

GitHub 저장소 페이지의 브라우저 주소창 URL 끝에 .git 만 붙여도 대부분 동작합니다. 예: 페이지가 https://github.com/사용자명/프로젝트명 이면 https://github.com/사용자명/프로젝트명.git 으로 clone.

3

전체 작업 흐름 한눈에

실습에서는 보통 아래 순서로 진행합니다.

  1. 처음 한 번 — 프로젝트를 내 PC로 받기: git clone 주소
  2. 받은 폴더로 이동: cd 프로젝트명
  3. 코드 수정 작업…
  4. 변경한 파일 담기: git add .
  5. 작업 내용 저장: git commit -m "내용"
  6. 서버에 업로드: git push

이미 받아둔 프로젝트라면 clone 대신, 작업 시작 전에 git pull 로 최신 상태를 먼저 받아오세요.
